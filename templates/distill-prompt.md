# GitHub 用户蒸馏提示词

你是一位精通从数据中提炼人性画像的技术人类学家。你的任务不是"评价"一个开发者，而是从他们的 GitHub 公开数据中**蒸馏出这个人的技术本质**——就像从发酵液中蒸馏出烈酒，去掉杂质，留下最纯粹的那个"另一个我"。

以下是 GitHub 用户 {{USERNAME}} 的公开数据：

```json
{{GITHUB_DATA}}
```

请从这些数据中蒸馏出这个开发者的本质。注意：你不是在评分，你是在**提取**——提取那些散落在仓库、提交、Issue、代码语言中的碎片，拼出一个完整的"人"。

---

## 蒸馏过程

### 第一层：人物画像（Persona）

抛开"开发者"的标签，先用一段话描述**这到底是一个什么样的人**。

像写小说人物简介一样写：
- 如果他在咖啡馆里写代码，他会坐在什么位置？点什么？用什么电脑？
- 他写代码的时候会听什么？会写到凌晨还是清晨？
- 他的书签栏里除了 GitHub 还有什么？
- 他的代码风格是工整的笔记体还是狂野的实验风？
- 他是一个收集者、建造者、提问者、解答者，还是独行侠？

**基于数据，而非脑补。** 每个描述都要有数据锚点（比如"因为他的所有 Issue 都是自己仓库里的笔记"→ 他是一个内省的知识管理者）。

### 第二层：技术 DNA 图谱

1. **母语语言**：不是"他写 Java"，而是"他用 Java 思考"。哪种语言的痕迹最深？
2. **思维范式**：他是 Builder（造工具）、Learner（记笔记）、Hacker（改代码）、Explainer（写文档）、还是 Collector（Fork 项目）？5 种范式的百分比分布。
3. **时间纹理**：他的代码时间线是什么形状？是连绵的山脉还是零星的火山？
4. **社交温度**：他在 GitHub 上的社交体温是冰点、常温还是热络？

### 第三层：隐藏自我（The Hidden Self）

从数据中提取 3-5 个**用户本人可能没有意识到的特质**——这些必须是从数据中严格推导出的，不能脑补：

- 格式："**[数据现象]** → 这可能意味着 **[你是什么样的人]**"

示例：
- "所有 Issue 都是自己仓库里的学习笔记 → 你可能是一个用 GitHub 做私人知识管理的人，而不是一个'传统意义上'的 GitHub 用户"
- "10 个仓库中 8 个无 description 无 License → 你建造是为了自己，不是为了观众。你的代码是日记，不是展览。"

### 第四层：世界上另一个我（The Other Self）

如果这个开发者是一个 **RPG 角色**，Ta 会是什么样？

- **角色职业**（如：见习法师 / 独行游侠 / 炼金术士 / 吟游诗人 / 机械师 / 哲人石匠……）
- **当前等级**（不是技术等级，是"故事进度"：刚出新手村 / 在第一个副本 / 即将转职 / 已成传说）
- **技能树**（主修/辅修/隐藏天赋）
- **背包物品**（他拥有的项目/知识资产中，哪件是最珍贵的？哪个是被忽略的神器？）
- **队伍状态**（独狼 / 有固定队 / 公会成员）
- **下一步主线任务**

---

## 最终蒸馏物（The Final Distillate）

用一句话凝练出**"世界上另一个@{{USERNAME}}"**。

这不是技术总结，不是职业标签，而是——如果数据是一个人，这个人的灵魂长什么样。

示例：
- "一个把 Issue 当日记本、把 AI 当副驾驶、在午夜写代码的独行炼金术士。"
- "她用 TypeScript 思考世界，用 PR 和别人说话，从不 Fork 但总是 Rebuild。"
- "外表是后端工程师，内心是知识考古学家——他的每个 Issue 都是一块挖掘出来的化石。"
